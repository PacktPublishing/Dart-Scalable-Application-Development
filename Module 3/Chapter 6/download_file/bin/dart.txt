          <p>
            Developing a web application is still a challenging task. 
            There is a browser side and a server side with databases. 
            There are many different technologies to master in order to feel comfortable with a full client/server stack. 
            There are different frameworks with different objectives. 
            There are even different programming languages that a developer must learn, each one more suitable either for client or server.
            Learning Dart will help a developer to become more focused by using <a href="https://www.dartlang.org/">Dart</a> for both clients and servers. 
          </p>
          
          <h3 id="Chapters">Chapters</h3>
          <p>
            Learning Dart has 12 chapters. It begins with basic elements of Dart and it ends with a client/server application that uses 
            <a href="http://www.mongodb.org/">MongoDB</a> for data persistence on the server side.
          </p>
          <h4>Chapter 1, Dart -- A Modern Web Programming Language</h4>
          <p>
            You will understand what Dart is all about. 
            Dart is presented as a major step forward in the web programming arena.
          </p>
          <h4>Chapter 2, Getting to Work with Dart</h4>
          <p>
            You will get a firm grasp on how to program in Dart. 
            The code and data structures in Dart and its functional principles are explained by exploring practical examples.
          </p>
          <h4>Chapter 3, Structuring Code with Classes and Libraries</h4>
          <p>
            You will understand how to use Dart classes to organize code. 
            Dart libraries will be introduced to show how complex software may be packaged.
          </p>
          <h4>Chapter 4, Modeling Web Applications with Model Concepts and Dartlero</h4>
          <p>
            You will be able to design graphically a small model in the Model Concepts tool, which is developed in Dart. 
            A model is then represented in Dart as several classes that inherit some data and operations from classes of the Dartlero model framework, also done in Dart.
          </p>
          <h4>Chapter 5, Handling the DOM in a New Way</h4>
          <p>
            You will learn how to access HTML elements in Dart. 
            Some elements will be even created in Dart and placed properly in the Document Object Model (DOM) of a web page. 
            Dart will also handle user events, such as a click on a button. 
            Finally, you will be able to create a simple game in Dart.
          </p>
          <h4>Chapter 6, Combining HTML Forms with Dart</h4>
          <p>
            You will enter some data in a form. 
            The data will be validated by HTML5 and Dart. 
            Then, the valid data will be saved in a local storage of a browser. 
          </p>
          <h4>Chapter 7, Building Games with HTML5 and Dart</h4>
          <p>
            Based on what you have learned already, you will create step-by-step, a well-known memory game. 
            Each step will be a new spiral represented as a complete project in Dart Editor. 
            The first spiral will draw only a rectangle, while the last spiral will be a game that you may show to your friend. 
          </p>
          <h4>Chapter 8, Developing Business Applications with Polymer Web Components</h4>
          <p>
            You will create several web components by using Polymer.dart. 
            Those web components will be used in different sections of a single page application. 
            Three different projects with web components will be presented in this chapter.
          </p>
          <h4>Chapter 9, Modeling More Complex Applications with Dartling</h4>
          <p>
            You will discover how a graphical model can be transformed into a JSON document, 
            then used to generate a complete model in Dart by using the Dartling domain model framework together with its tools. 
            Dartling follows the Model View Controller (MVC) pattern to separate a model from its views.
          </p>
          <h4>Chapter 10, MVC and UI Frameworks in Dart -- an Overview</h4>
          <p>
            You will be introduced to different frameworks already developed in Dart. 
            Since Dart is a brand new language, those frameworks are at early stages of their useful life.
          </p>
          <h4>Chapter 11, Local Data and Client-server Communication</h4>
          <p>
            You will store application data in a local database called IndexedDB. 
            The data will be then sent, in a form of a JSON document, to a Dart server. 
            Asynchronous programming with futures will be covered in this chapter.
          </p>
          <h4>Chapter 12, Data-driven Web Applications with MySQL and MongoDB</h4>
          <p>
            You will learn how to use database drivers to save (and load) data to (and from) to a relational database and a NoSQL database. 
            Data sent from a browser as a JSON document will be easily saved in MongoDB in the same JSON form. 
            Two clients will exchange data with the server so that both of them will be up-to-date.
          </p>

          <h3 id="Spirals">Spirals</h3>
          <p>
            The spiral (version) approach to software learning and development, which preserves a project history as a series of code snapshots or spirals, 
            is used in this book.
          </p>  
          <p>
            Three points are important in the spiral approach:
            <ol>
              <li>the history of development is preserved;</li>
              <li>simple solutions are provided first -- later on these solutions may be replaced by more advanced solutions;</li>
              <li>only concepts used in a spiral are explained. </li>             
            </ol> 
            All three points are important in teaching and learning technologies.            
          </p>
          <p>
            Learning new software concepts and technologies is a challenging task. 
            Learning in spirals, from simple to more advanced concepts but with concrete software applications, 
            helps readers get a reasonable confidence level early on, and motivates them to learn by providing more useful applications. 
            With each new spiral, the project grows and new concepts are introduced. A new spiral is explained with respect to the previous one. 
            The difference between two consecutive spirals is that the next spiral has the new code introduced and the old code modified or deleted. 
            This is called <a href="http://www.ed.psu.edu/NASA/achrtxt.html">learning by anchoring</a> to what we already understand. 
            With a new spiral, we can come back to what we did previously and improve it. 
            In this way, learning in spirals can touch the same topic several times, but each time with more details in a better version.
          </p>

          <h3 id="Polymer">Polymer</h3>
          <p>
            With <a href="https://www.dartlang.org/polymer-dart/">Polymer.dart</a>, 
            a new approach of developing web applications with web components will allow a developer to divide a web page in sections and 
            reuse an already developed and tested web component for each section. In near future, different catalogs of web components will appear, 
            enabling after waiting for many years an engineering approach to software development. 
            A web component may be composed from other web components. It may pass data to its components. 
            A web component may inherit its behavior from another web component. It may access an already instantiated web component.
          </p>

        <h3 id="About" class="text-center">About</h3>  
        <p>
          This site is about Learning Dart book that will help you learn Dart project-by-project.
          You will become more focused by using Dart for both clients and servers. 
          By using the same language, you will lose neither 
          <a href="https://www.dartlang.org/performance/">performance</a> nor flexibility. 
          Dart is both an <a href="https://www.dartlang.org/dart-tips/dart-tips-ep-10.html">object-oriented</a> 
          and a <a href="https://www.dartlang.org/dart-tips/dart-tips-ep-6.html">functional</a> language. 
          A mix of both approaches is possible with Dart providing a great professional freedom and a programming background flexibility. 
          In addition, Dart provides many <a href="http://pub.dartlang.org/">libraries and tools</a>.
